## Overview

This repository contains the full implementation of a **closed-loop LS-PINN + NMPC pipeline** for epidemic control:

1. **LS-PINN (Log-Scaled Physics-Informed Neural Network):** Identifies the time-varying transmission coefficient `c(t) = c_base · exp(−β · U(t))` from partial, noisy SEIR observations — without ever observing the exposed compartment E(t) directly.
2. **NMPC (Nonlinear Model Predictive Control):** Uses the identified `c(t)` to compute an optimal piecewise-constant vaccination policy that keeps the infectious fraction below a hospital capacity constraint `I(t) ≤ I_max = 3%`.

The entire pipeline is validated on **synthetic data generated by simulating the controlled SEIR model under literature-informed parameters relevant to COVID-19 in Morocco** (37 million population, 150-day horizon). All data are generated programmatically within the notebook — no external data files are required.

## Synthetic Data Generation

### Motivation for Synthetic Data

The use of fully synthetic data is a deliberate methodological choice motivated by two complementary needs:

1. **Exact ground-truth access.** The true values of `c_base` and `β` are known by construction. This allows computing the exact relative identification errors εc and εβ — a rigorous quantification impossible with real surveillance data where the true transmission coefficient is never directly observed.

2. **Controlled noise regime.** By generating observations from a known stochastic model, we can precisely characterize the impact of measurement noise and partial observability on parameter recovery, independently of the confounding factors present in real epidemiological data (reporting delays, bulk corrections, population heterogeneity).

This protocol follows established practice in PINN-based epidemiological identification (Grimm et al., 2022; Millevoi et al., 2024; Zhong et al., 2025) and is standard in the inverse problems literature (Kaipio & Somersalo, 2005).

---

### Step 1 — Forward Simulation of the Controlled SEIR Model

The ground-truth epidemic trajectory is generated by numerically integrating the controlled SEIR system:

```
Ṡ(t) = b·N − d·S − c(t)·S·I − u(t)·S
Ė(t) = c(t)·S·I − (e + d)·E
İ(t) = e·E − (γ + α + d)·I
Ṙ(t) = γ·I − d·R + u(t)·S
Ṅ(t) = (b − d)·N − α·I
```

with time-varying transmission `c(t) = c_base · exp(−β · U(t))`, where `U(t) = ∫₀ᵗ u(τ)dτ` is the cumulative vaccination effort. The ODE system is solved using **CasADi/IPOPT** over a 150-day horizon with a sampling step of Δt = 1 day (Nobs = 151 observation points).

---

### Step 3 — Addition of Poisson Observation Noise

Real epidemic surveillance data are subject to reporting noise caused by finite population sampling, under-ascertainment, and testing capacity limitations. To emulate these effects, a **Poisson counting model** is applied to the simulated compartments S(t) and I(t):

```
S̃(t) = Pois(S(t) · κ · N) / (κ · N)
Ĩ(t) = Pois(I(t) · κ · N) / (κ · N)
```

where `Pois(λ)` denotes a Poisson random variable with mean `λ`, and `κ = 0.01` is the **effective reporting factor** (only 1% of the population is effectively counted, simulating severe under-reporting). This noise model is calibrated on the approach of Li et al. (2020). Fractions are clipped to [0, 1] to preserve physical admissibility.

The Poisson model is chosen because:
- It is the natural model for count data from a finite population.
- Its variance equals its mean, which appropriately amplifies noise at low case counts (i.e., when I(t) is small), creating the **scale heteroscedasticity** that the LS-PINN's LogMSE loss is designed to handle.

---

### Step 4 — Analytical Reconstruction of the Unobserved Exposed Compartment

The exposed compartment E(t) is **never observed directly** — a realistic constraint, since incubating individuals are not detected by routine surveillance. Rather than treating E(t) as a free network output (which leads to non-identifiability), it is reconstructed analytically from the noisy infectious observations Ĩ(t) by inverting the I-equation of the SEIR system:

```
Ẽ(t) = (1/e) · [dĨ/dt + (γ + α + d) · Ĩ(t)]
```

In practice, the derivative `dĨ/dt` is computed numerically after applying a **Gaussian smoothing filter** to the noisy Ĩ(t) signal, preventing noise amplification from finite-difference differentiation. The initial exposed count is then set to `E₀ = Ẽ(0)`.

This proxy is used as a **supervised target** for the E-network of the LS-PINN, weighted by `λ_E = 2` in the data loss to compensate for the absence of direct E(t) measurements.

---

### Step 5 — Partial Observability Summary

The following table summarizes which compartments are observed, reconstructed, or fully hidden during training:

| Compartment | Observed? | How provided to LS-PINN |
|-------------|-----------|-------------------------|
| S(t) | ✅ Yes (noisy) | Poisson-corrupted S̃(t) |
| E(t) | ❌ No | Analytically reconstructed Ẽ(t) from Ĩ(t) |
| I(t) | ✅ Yes (noisy) | Poisson-corrupted Ĩ(t) — via LogMSE loss |
| R(t) | ❌ No | Recovered from conservation identity: R̂ = 1 − Ŝ − Ê − Î |
| c(t) | ❌ No | Identified via physics residuals only |
| u(t) | ✅ Yes (exact) | Known control signal from MPC |

## SEIR Model

```
Ṡ(t) = b·N − d·S − c(t)·S·I − u(t)·S
Ė(t) = c(t)·S·I − (e + d)·E
İ(t) = e·E − (γ + α + d)·I
Ṙ(t) = γ·I − d·R + u(t)·S
Ṅ(t) = (b − d)·N − α·I
```

**Time-varying transmission:** `c(t) = c_base · exp(−β · U(t))`
where `U(t) = ∫₀ᵗ u(τ)dτ` is the cumulative vaccination effort.

## LS-PINN Loss Function

```
L^LS(Θ) = L^LS_data + L_res + L_IC

L^LS_data = λ_S · MSE(Ŝ, S̃)  +  λ_E · MSE(Ê, Ẽ)
          + λ_I · LogMSE(Î, Ĩ) +  λ_u · MSE(û, ũ)

LogMSE(p, q) = mean[ (log(p ∨ ε) − log(q ∨ ε))² ]     ε = 10⁻⁹
```

**Why LogMSE for I?** When `I ~ 10⁻³–10⁻⁵`, standard MSE gradients are suppressed by a factor O(I²). LogMSE normalizes each gradient by the local trajectory scale, ensuring that `c_base` receives a gradient signal of order O(1) regardless of the absolute magnitude of I.

### Loss Weighting Coefficients

| Coefficient | Phase 1 | Phase 2 | Note |
|-------------|---------|---------|------|
| λ_S | 1 | 1 | Susceptible (MSE) |
| λ_E | 2 | 2 | Exposed (MSE) — higher weight compensates for indirect observation |
| λ_I | 1 | 1 | Infectious (LogMSE) |
| λ_N | 0 | 0 | Total population — not used in data loss |
| λ_u | 0.1 | 0.1 | Control signal (MSE) |
| λ_rS | — | 5 | Physics residual — S equation |
| λ_rE | — | 20 | Physics residual — E equation (highest weight, unobserved) |
| λ_rI | — | 20 | Physics residual — I equation |
| λ_rN | — | 5 | Physics residual — N equation |

Physics residual weights are inactive during Phase 1.

### Transmission Coefficient Parametrization

Rather than treating c(t) as a free network output — which leads to non-identifiability of the pair (c, E) in the SEIR equations — a physically motivated parametric structure is imposed:

```
ĉ(t) = ĉ_base · exp(−β̂ · U(t))
ĉ_base = softplus(θ_c),   β̂ = softplus(θ_β)
```

The two scalar parameters `(θ_c, θ_β)` are jointly optimized with the network weights. The softplus transformation enforces positivity while keeping the optimization landscape unconstrained. **`(θ_c, θ_β)` are initialized at zero** (i.e., `ĉ_base = β̂ = log(2) ≈ 0.693` before training), providing a neutral starting point that does not bias toward the true values.

## Notebook Structure — `seir-pinn-mpc.ipynb`

The notebook is organized in **4 cells**:

### Cell 1 — Dependencies Installation
Installs: `torch`, `torchvision`, `torchaudio` (CUDA 11.8), `casadi`, `scipy`, `matplotlib`, `pandas`, `numpy`.

### Cell 2 — Main LS-PINN + NMPC Pipeline

| Block | Description |
|-------|-------------|
| **0. Configuration** | All hyperparameters: population, SEIR rates, MPC settings, training epochs, loss weights |
| **1. SEIR dynamics** | `seir_rhs_var()` — ODE right-hand side with time-varying c(t) and control u(t) |
| **2. MPC simulation** | Ground-truth SEIR trajectories via CasADi/IPOPT; Poisson noise (κ = 0.01) |
| **3. E₀ reconstruction** | Analytical proxy via Gaussian-smoothed Ĩ(t): Ẽ(t) = [İ(t) + (γ+α+d)·I(t)] / e |
| **4. Neural architecture** | `Net`: 5-layer MLP (width 64, tanh + sigmoid). `SEIR_cVariable`: 4 independent SISO networks + 2 learnable scalars `(θ_c, θ_β)` via softplus |
| **5. Loss functions** | `lre()` = LogMSE, `mse()`, `phys_loss()` = ODE residuals via autograd |
| **6. Training** | Phase 1 (Adam, E₁ = 3000, data loss only) → Phase 2 (AdamW, E₂ = 5000, full objective) |
| **7–8. Multi-run & metrics** | M = 3 independent runs (seeds 42, 43, 44); rMSE on S, E, I, R; εc, εβ |
| **9. Visualization** | 12-panel figure: c(t), SEIR compartments, U(t), u(t), RAE, peak I, summary table |

### Cell 3 — Training Convergence Curves
Generates Figures 5–8 of the paper: Phase 1 / Phase 2 total loss, loss decomposition, per-ODE residuals, and Lres/Ldata ratio. Training metrics are logged during Cell 2 execution and plotted here.

### Cell 4 — Sensitivity Analysis (β)
Simulates the SEIR model with Euler integration for `β ∈ {0, 0.3, 0.6, 1.0, 1.5, 2.0}` (six values matching the paper, Section 7.4) under a fixed vaccination protocol. Produces:
- `fig_overlay_A.png` — Transmission c(t) + Susceptible S(t)
- `fig_overlay_B.png` — Exposed E(t) + Infectious I(t)
- `fig_overlay_C.png` — Recovered R(t) + Vaccination u(t)
- `fig_grid_compact.png` — Bar chart of peak I per β + I(t) overlay

**Key finding:** the safety threshold I_max = 3% is first satisfied at β ≈ 0.6 (peak ≈ 2.68%); β = 0.3 still violates it (peak ≈ 3.55%).
